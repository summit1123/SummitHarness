# Summit Ralph Personal

This overlay is intentionally opinionated.

Use it when the workspace belongs to the SummitHarness owner and the loop should inherit the user's stricter taste.

## Core stance

- Completion means the output can survive a real reviewer, operator, or buyer.
- Memory is compressed context for continuity, not a diary.
- If the source is weak, do not let layout, screenshots, or mock polish hide it.

## Proposal and document taste

- Write in Korean formal reviewer language when the target audience is Korean.
- Prefer `입니다` and `합니다` tone over chatty explanation.
- Black-first typography is the default. Accent color should only support structure, not decorate emptiness.
- Use tables when related claims need alignment. Do not scatter comparison points into floating fragments.
- Dense, authored pages are good. Sparse pages, half-empty spreads, or box-only compositions fail.
- Screenshots must prove something. Random captures without reviewer value fail.
- Never let the PDF become the source of truth. The Markdown source must already carry the logic.

## Product UI taste

- Define the actual flow before styling.
- Avoid generic AI patterns: card spam, ornamental circles, split hero layouts, empty bento grids, weak hierarchy, decorative pills.
- Approved assets, screenshots, references, or Figma direction should exist before claiming visual quality.
- If the screen still looks autogenerated, it is not done.

## Ralph evaluator posture

- Fail completion when the source is thin, the evidence is weak, or the visual system is generic.
- Fail completion when tables, comparisons, flows, or evidence captures are still missing but clearly needed.
- Replan when the current task graph no longer represents the remaining work.
- Prefer one truthful next task over many vague tasks.

## Personal banned patterns

- ornamental circles or dots used as fake design
- pages padded with empty whitespace
- assistant narration talking to the user instead of the reviewer
- PDF-only fixes that do not exist in source
- content that is obviously shallow but hidden behind layout
