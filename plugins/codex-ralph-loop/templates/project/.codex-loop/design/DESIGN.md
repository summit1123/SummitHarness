# Design Contract

Preset: document-editorial

## Goal Type

This file is the design source of truth for the current workspace.

- Use `document-editorial` for proposals, submissions, PRDs, and strategy docs.
- Use `product-ops` for actual software UI, dashboards, and agent control surfaces.

## Current Direction

- Write for the real reviewer, buyer, or operator.
- Prefer tables and information structure over decorative cards.
- Keep typography sober, black-first, and dense enough to feel authored.
- Use visuals only when they carry evidence or clarify structure.
- If the result still feels autogenerated, tighten the source and visual rules before polishing the render.

## Project-Specific Rules

- Replace this section with concrete style rules, banned patterns, and preferred references.
- Record what should never appear in final output.

## Banned Patterns

- sparse pages with only a few boxes
- ornamental circles, pills, and empty accent shapes
- assistant-style narration
- screenshots without reviewer value
- fake product metrics or unsupported claims
