# Summit Intake

Use this skill before task seeding or long autonomous runs when the goal is still fuzzy.

## Workflow

1. Read `.codex-loop/intake/QUESTIONNAIRE.md`, `.codex-loop/intake/ANSWERS.md`, and `.codex-loop/intake/APPROVAL.md`.
2. Ask the smallest number of role-specific questions needed to make the goal trustworthy.
3. Capture final answers in `ANSWERS.md` instead of keeping them only in chat.
4. Lock the approved goal, scope, and evidence bar in `APPROVAL.md`.
5. Do not start autonomous planning until the approval gate is explicit.

## Output bar

- The requester, reviewer, and working mode are unambiguous.
- Out-of-scope work is visible.
- The evidence bar for COMPLETE is specific enough to prevent fake finishes.
- Brainstorming and planning can proceed without re-asking the same basic questions.
